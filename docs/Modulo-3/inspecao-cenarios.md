# Inspeção dos Cenários

</center>

## Metodologia

A verificação dos cenários ([versão 1.1](/Modulo-2/cenario.md)) foi feita pela técnica de inspeção. Foram elaboradas uma série de
perguntas de "sim" ou "não" que refletem as características de um bom cenário. Tais respostam acarretam impactos
positivos ou negativos.

Em seguida, foram analisados os 35 cenários, respondendo para cada um as perguntas, preenchendo uma tabela de avaliação
com um "X" caso a resposta seja "sim" e " " caso seja "não".

---

## Checklist para Verificação dos Cenários

<div style="margin: 0 auto; width: fit-content;">

| ID | Pergunta                                                                          |
|----|-----------------------------------------------------------------------------------|
| 1  | O cenário possui um título claro e descritivo?                                    |
| 2  | O cenário possui um objetivo ou meta claramente definidos?                        |
| 3  | O cenário possui um contexto bem definido (contendo local, tempo e pré-condição)? |
| 4  | O cenário identifica os atores envolvidos?                                        |
| 5  | O cenário descreve os recursos necessários para sua execução?                     |
| 6  | O cenário possui episódios claramente definidos?                                  |
| 7  | O cenário especifica as restrições aplicáveis?                                    |
| 8  | O cenário considera exceções ou situações de erro?                                |
| 9  | O cenário possui ligações (hyperlinks) com os léxicos?                            |

</div>

<center>

Fonte: [Cássio Reis](https://github.com/csreis72), 2024

</center>

## Problemas Encontrados

- Falta de Restrições: Não foram definidas restrições para nenhum cenário;

- Ausência de tempo e local no contexto: Vários cenários não possuem informações sobre tempo e local nos contextos;

- Recursos incompletos: Foi identificada a falta de recursos necessários para a execução de alguns cenários, conforme
  detalhado na tabela abaixo:

<div style="margin: 0 auto; width: fit-content;">

| Cenário                                         | Problema                                                              |
|-------------------------------------------------|-----------------------------------------------------------------------|
| C07 Cadastrar Imóvel para Alugar                | Falta de recursos relacionados às informações do imóvel.              |
| C11 Acessar Contrato de Aluguel                 | Ausência do contrato como recurso.                                    |
| C14 Enviar Documentos para Locação              | Documentos necessários não estão listados como recurso.               |
| C15 Cancelar Proposta de Aluguel                | Falta da proposta enviada como recurso.                               |
| C16 Visualizar Fotos do Imóvel                  | Fotos do imóvel não estão listadas como recurso.                      |
| C17 Receber Confirmação de Visita               | Confirmação de visita não está listada como recurso.                  |
| C19 Reagendar Visita                            | Visita agendada não está listada como recurso.                        |
| C23 Cadastrar Forma de Pagamento                | Informações de pagamento necessárias não estão listadas como recurso. |
| C25 Solicitar Segunda Via de Boleto             | Boleto necessário para a solicitação não está listado como recurso.   |
| C28 Receber Propostas de Aluguel                | Propostas recebidas não estão listadas como recurso.                  |
| C30 Verificar Status de Proposta (Proprietário) | Proposta não está listada como recurso.                               |
| C35 Atualizar Fotos do Imóvel (Proprietário)    | Fotos para atualização não estão listadas como recurso.               |
